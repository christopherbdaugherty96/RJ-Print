# RJ Print Scope And Delivery Plan

Status: planning baseline

## Purpose

Define the first realistic website scope for RJ Print so the project does not drift into automation, e-commerce complexity, or unclear custom work before the client approves the basics.

## Project type

Client website build for a 3D print shop.

## Version 1 goal

Launch a clear, professional website that explains RJ Print, shows product categories/examples, and gives customers a simple way to request a custom print or contact the business.

## Recommended v1 scope

### Pages

- Home
- Shop or Product Gallery
- Custom Print Request
- About RJ Print
- FAQ
- Shipping / Pickup Info
- Contact

### Content sections

- Hero with clear offer
- Product lanes: ready-made, made-to-order, custom requests
- Featured product/category section
- Custom request process
- Gallery or product examples
- Trust/expectation section
- FAQ
- Contact/request call to action

### Functional requirements

- Mobile-friendly layout
- Working contact or quote request form
- Clear email/phone fallback if form fails
- Basic SEO title/description
- Social preview image/text if practical
- No fake reviews or fake proof
- No unsupported automation claims

## Out of scope for v1

- Shopify checkout unless specifically approved
- Custom cart
- Payment processing through the website
- Customer accounts
- Printer queue automation
- Shopify-to-printer automation
- File upload system unless a simple tool is selected
- Real-time order tracking
- Automated quoting
- Advanced inventory system

## Client responsibilities

Client should provide:

- Logo or approval for text-based logo
- Contact information
- Product list
- Product photos
- Product prices
- Current inventory if ready-made items are shown
- Shipping/pickup rules
- Custom request boundaries
- Approval before launch

## Builder responsibilities

Auralis / website builder should provide:

- Website structure
- Homepage and page copy draft
- Product/category organization
- Contact/request form setup
- Mobile and desktop checks
- Basic launch checklist
- Handoff notes

## Approval checkpoints

1. Client approves business positioning.
2. Client approves product lanes and categories.
3. Client approves product prices or quote labels.
4. Client approves custom request process.
5. Client approves homepage draft.
6. Client approves final launch.

## Delivery definition

The v1 project is considered delivered when:

- agreed pages are built
- contact/request path works
- client has reviewed and approved final site
- mobile and desktop checks pass
- launch checklist is reviewed
- handoff notes are provided

## Later phases

### Phase 2

- Add more products
- Add stronger gallery
- Add customer proof
- Improve SEO and local search
- Add analytics review

### Phase 3

- Add Shopify checkout if needed
- Add custom quote/file upload workflow
- Add inventory tracking
- Add business/bulk order page

### Phase 4

- Explore printer/order automation only after manual workflow is proven.
