# RJ Print Client Handoff README

Status: client handoff guide / requires client confirmation

## Purpose

This file explains how to read the RJ Print planning repo and what RJ Print still needs to confirm before a website can be built or launched.

## What this repo is

This repo is a planning and business-launch binder for RJ Print, a 3D print shop website project.

It organizes:

- business basics
- product categories
- pricing and margin planning
- custom print request workflow
- policy needs
- Shopify/readiness questions
- launch readiness
- risks and blockers
- product inventory data collection

## What this repo is not yet

This repo is not a finished public website.

It does not yet include:

- production website implementation
- live storefront
- confirmed Shopify checkout
- confirmed tax, legal, insurance, or payment readiness
- confirmed product inventory
- confirmed shipping or pickup policy
- printer automation
- Shopify-to-printer automation
- automatic custom print checkout

Printer automation is future-only and is not part of version 1.

## Placeholder rule

Anything marked `TBD`, `example`, `draft`, `to be confirmed`, `requires client confirmation`, `placeholder`, or `not finalized` is not launch-ready by default.

Before launch, each placeholder must be replaced with real RJ Print information, approved as intentionally generic, or removed.

Do not invent missing facts. Use `TBD`, `to be confirmed`, or `requires client confirmation` until RJ Print confirms the detail.

## Most important next action

Fill out `data/product_inventory_template.csv` with real RJ Print product data.

Start with the first 5-10 products RJ Print would actually want customers to see or request.

## Launch rule

Do not launch or publicly promote the website until the client has approved product categories, product prices, contact flow, pickup/shipping expectations, policy expectations, and launch scope.

A quote/contact-first website can be built before full checkout if expectations are clear and the site does not imply unsupported checkout, legal/tax/insurance readiness, automated fulfillment, or guaranteed custom print acceptance.
