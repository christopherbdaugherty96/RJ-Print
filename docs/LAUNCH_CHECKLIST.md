# RJ Print Launch Checklist

Status: planning baseline

## Purpose

Track what must be true before the RJ Print website is treated as ready for public promotion.

## Business setup

- [ ] Confirm public business name.
- [ ] Confirm logo or text-brand fallback.
- [ ] Confirm public contact email.
- [ ] Confirm public phone if used.
- [ ] Confirm service area / online shipping scope.
- [ ] Confirm pickup/shipping policy.
- [ ] Confirm refund/replacement policy.

## Product setup

- [ ] Product list created.
- [ ] Products separated into ready-made, made-to-order, custom, and bulk lanes.
- [ ] First featured products selected.
- [ ] Product photos collected.
- [ ] Product prices confirmed.
- [ ] Product categories confirmed.
- [ ] Material and production-time expectations written.
- [ ] Margin review completed for featured products.
- [ ] Bad-margin or unclear products held back from launch.

## Custom request setup

- [ ] Custom request process written.
- [ ] Custom request form questions confirmed.
- [ ] Quote review process confirmed.
- [ ] Deposit/payment expectations confirmed if applicable.
- [ ] File/photo submission workflow confirmed.
- [ ] Decline/boundary rules confirmed.

## Website pages

- [ ] Home
- [ ] Shop or Gallery
- [ ] Ready-Made Prints
- [ ] Made-to-Order Prints
- [ ] Custom Print Request
- [ ] Business / Bulk Orders if included in v1
- [ ] About
- [ ] FAQ
- [ ] Shipping / Pickup Info
- [ ] Contact

## Website quality

- [ ] Homepage explains the business clearly.
- [ ] Main call to action is visible above the fold.
- [ ] Product lanes are easy to understand.
- [ ] Contact/custom request path works.
- [ ] Mobile layout checked.
- [ ] Desktop layout checked.
- [ ] Product images load.
- [ ] No placeholder text remains.
- [ ] No fake reviews or fake claims are used.
- [ ] Production timelines are not overstated.
- [ ] Custom work is not presented as instant checkout unless reviewed.

## Technical setup

- [ ] Domain decision confirmed.
- [ ] Hosting/deployment decision confirmed.
- [ ] Contact form destination confirmed.
- [ ] Form tested from desktop.
- [ ] Form tested from phone.
- [ ] SEO title and description added.
- [ ] Social preview image/text checked.
- [ ] Sitemap/robots considered if custom site.
- [ ] Analytics decision made.

## Client approval

- [ ] Client reviewed homepage.
- [ ] Client reviewed product categories.
- [ ] Client reviewed product prices.
- [ ] Client reviewed custom request process.
- [ ] Client reviewed policies.
- [ ] Client approved launch.

## Post-launch

- [ ] Confirm live site loads.
- [ ] Confirm mobile live site works.
- [ ] Submit test contact/custom request.
- [ ] Track first inquiry.
- [ ] Track first sale/request if it happens.
- [ ] Collect feedback from client.
- [ ] Update site based on real customer questions.

## Boundary

Do not launch as a mature e-commerce system unless checkout, payment, tax, shipping, inventory, and fulfillment workflow are actually verified.
