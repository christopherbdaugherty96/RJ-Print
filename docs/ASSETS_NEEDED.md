# RJ Print Assets Needed

Status: planning baseline

## Purpose

Track what assets are needed before the RJ Print website can be built and launched cleanly.

## Brand assets

- [ ] Logo
- [ ] Brand colors
- [ ] Font preferences if any
- [ ] Tagline if any
- [ ] Business contact email
- [ ] Business phone if public
- [ ] Social links

## Product photos

Needed for each featured product:

- [ ] Main product photo
- [ ] Secondary angle
- [ ] Scale/context photo
- [ ] Color/material variation photos if applicable
- [ ] Lifestyle/use photo if available

## Product information

Needed for each product:

- [ ] Product name
- [ ] Product category
- [ ] Ready-made, made-to-order, custom, or bulk lane
- [ ] Price or starting price
- [ ] Materials
- [ ] Colors
- [ ] Size/dimensions
- [ ] Production time
- [ ] Current inventory count if ready-made
- [ ] Shipping/pickup note
- [ ] Customization options

## Business content

- [ ] About RJ Print story
- [ ] What makes the prints different
- [ ] How custom requests work
- [ ] Turnaround time expectations
- [ ] Pickup/shipping explanation
- [ ] Refund/replacement policy
- [ ] FAQ answers

## Custom request materials

- [ ] Example custom request photos
- [ ] Before/after or idea-to-print examples
- [ ] Quote workflow approval language
- [ ] File/reference upload method
- [ ] Custom request boundaries

## Website proof/trust

- [ ] Real customer reviews if approved
- [ ] Finished-product gallery
- [ ] Photos of printer/workspace if desired
- [ ] Social media examples
- [ ] Local pickup/shipping proof if relevant

## Technical assets

- [ ] Domain decision
- [ ] Hosting decision
- [ ] Contact form destination
- [ ] Shopify or no-Shopify decision for v1
- [ ] Analytics setup decision
- [ ] File upload/form tool decision if custom uploads are needed

## Priority assets for first version

Minimum required before v1 launch:

1. Logo or simple text brand mark
2. Public contact method
3. 5-10 strong product photos
4. Product names and prices
5. Custom request process
6. Shipping/pickup expectations
7. About/FAQ basics

## Notes

Do not wait for perfect assets if the client needs a simple first version. Launch can start with the strongest products and expand after real photos, costs, and margins are confirmed.
