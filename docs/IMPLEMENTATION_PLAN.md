# RJ Print Implementation Plan

Status: planning baseline

## Purpose

Define the order for turning the RJ Print planning docs into an actual website.

## Current phase

Discovery and planning.

Do not begin full website implementation until the client provides enough product, asset, pricing, and contact information for a useful first version.

## Phase 1 — Discovery

Required inputs:

- Client profile confirmed
- Product list collected
- Product photos collected or scheduled
- Public contact info confirmed
- Pickup/shipping policy confirmed
- Product lanes confirmed
- Custom request process confirmed
- Website v1 scope approved

Relevant files:

- `docs/CLIENT_PROFILE.md`
- `docs/QUESTIONS_FOR_CLIENT.md`
- `docs/ASSETS_NEEDED.md`
- `data/product_inventory_template.csv`

## Phase 2 — Scope approval

Required decisions:

- Quote/contact-first or checkout-enabled v1
- Final page list
- Number of products/categories included
- Whether file uploads are included
- Domain/hosting decision
- Price/timeline agreement

Relevant files:

- `docs/SCOPE_AND_DELIVERY_PLAN.md`
- `docs/CLIENT_QUOTE_AND_SCOPE_TEMPLATE.md`
- `docs/LAUNCH_CHECKLIST.md`

## Phase 3 — Copy and content

Create or finalize:

- Homepage copy
- About copy
- Product/category descriptions
- Custom request copy
- FAQ copy
- Shipping/pickup copy
- Contact form copy

Relevant files:

- `docs/WEBSITE_COPY_DRAFT.md`
- `docs/WEBSITE_PLAN.md`
- `docs/CUSTOM_PRINT_REQUEST_FLOW.md`

## Phase 4 — Website build

Recommended v1 pages:

- Home
- Product Gallery or Shop Preview
- Custom Print Request
- About RJ Print
- FAQ
- Shipping / Pickup Info
- Contact

Minimum technical requirements:

- Mobile-friendly layout
- Working contact/custom request form
- Basic SEO title/description
- Clear fallback contact method
- No fake reviews or unsupported claims
- No printer automation claims

## Phase 5 — QA

Check:

- Desktop layout
- Mobile layout
- Navigation
- Images
- Contact form
- Form notification email
- SEO metadata
- Social preview if applicable
- Policy/expectation language
- No placeholder copy
- Client approval

Relevant file:

- `docs/LAUNCH_CHECKLIST.md`

## Phase 6 — Launch

Launch only after:

- Client approves final draft
- Contact path works
- Product/category info is approved
- Pickup/shipping expectations are clear
- Final URL is confirmed

## Phase 7 — Handoff

Provide:

- Live site link
- What was built
- How contact requests work
- How updates should be requested
- Known limitations
- Recommended next steps

Relevant file:

- `docs/HANDOFF_NOTES_TEMPLATE.md`

## Later phases

Only after v1 is proven:

- Shopify checkout
- More product pages
- File uploads
- Analytics review
- Business/bulk quote page
- Inventory workflow
- Printer/order automation planning

## Current blocker

Real client data is required before build.

Priority input:

1. Actual products
2. Product photos
3. Product prices
4. Contact info
5. Pickup/shipping decision
6. Custom quote rules
