# RJ Print

Client website and business-planning project for a 3D print shop.

## Project role

RJ Print is a client build managed through Auralis Digital's website-business workflow.

This repo is for organizing the client's business structure, website plan, product categories, pricing/margin system, custom print request process, scope, launch readiness, business legitimacy, policy readiness, risk tracking, and eventual website implementation.

## Current client problem

RJ Print is already selling 3D printed products, but the business needs clearer structure around:

- product categories
- ready-made vs made-to-order products
- custom print requests
- pricing and margins
- material/print-time/labor costs
- shipping or local pickup expectations
- customer request flow
- website pages and launch checklist
- business legitimacy and policy readiness

## Core website lanes

1. Ready-Made Prints — items already printed and available now.
2. Made-to-Order Prints — known products printed after purchase/request.
3. Custom Print Requests — quote-based jobs from a customer idea, photo, measurement, or need.
4. Business / Bulk Orders — larger or repeat orders for businesses, gifts, displays, signs, or branded items.

## Boundary

Do not add automated printer integration, Shopify-to-printer automation, custom backend checkout, or printer queue logic until the product categories, pricing/margin system, client approval path, and first website scope are defined and approved.
